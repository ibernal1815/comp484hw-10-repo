Message Logging
console.info()
Outputs an informational message. Renders with a blue icon in DevTools and is filterable under the Info level.
console.warn()
Outputs a warning message. DevTools highlights it in yellow with a triangle icon. Use for non-fatal issues that need attention.
console.error()
Outputs an error message with a full stack trace attached. Rendered in red. Increments the error counter shown in the DevTools toolbar.
console.table()
Renders an array of objects as a sortable, interactive table inside DevTools. Each object key becomes a column. The static preview below shows the data being passed in.
nameroleclearance
AliceSOC AnalystL1
BobThreat HunterL3
CarolPen TesterL2
console.group()
Nests related messages under a collapsible group header in DevTools. Closed with console.groupEnd(). Useful for organizing request or event data into readable hierarchies.
console.log() with %c styling
The %c format specifier applies inline CSS to console output. Pass the style string as a second argument. Only supported in Chromium-based browsers.
View Messages Logged by the Browser
404 Network Error
Requesting a URL that does not exist returns a 404 status. The browser logs a red network error to the Console automatically. It also appears in the Network tab with status 404.

TypeError
Accessing a property or calling a method on null or undefined throws a TypeError at runtime. The browser logs it with a full stack trace pointing to the source line.
Waiting for trigger...
Where to look: Console tab shows the red TypeError entry. Click the source link on the right side of the entry to jump to the offending line in the Sources panel.
Violation
Long-running synchronous tasks that block the main thread beyond roughly 50ms generate a Violation warning. Alternating DOM reads and writes in a tight loop also triggers forced reflow violations. Set the Console log level to Verbose to see these entries.
Waiting for trigger...
Where to look: Open Console, set the log level dropdown to Verbose. After triggering, look for yellow [Violation] entries labeled "Forced reflow" or "Long running JavaScript task."

Debugging
Step 1
Reproduce the Bug
The function calcTotalScore() is supposed to sum the numeric score from each security alert and return a total. Instead it returns NaN because it reads alert.severity (a string like "high") instead of alert.score (a number). Click the button to reproduce the broken output.
Output will appear here...
What to look for: The result printed to the Console will be NaN instead of a number. This is the bug we will isolate using the Sources panel.
Step 2
Get Familiar with the Sources UI
Open DevTools (F12) and click the Sources tab. The panel has three panes working together.
File Navigator
Left pane. Lists every file the page loaded — HTML, CSS, JS. Click script.js here to open it in the editor pane.
Code Editor
Center pane. Shows the source code of the selected file. Click any line number to set a breakpoint on that line. A blue marker appears when a breakpoint is active.
Debugger Pane
Right pane. Shows Breakpoints, Scope variables, Watch Expressions, and the Call Stack when execution is paused.
Screenshot this: Open Sources and click script.js in the File Navigator. Take a screenshot showing all three panes visible at the same time.
Step 3
Set a Line-of-Code Breakpoint
In the Sources panel with script.js open, scroll to the function calcTotalScore(). Click the line number next to this line:
total += alert.severity;
A blue marker will appear on that line number confirming the breakpoint is active. It also shows up in the Breakpoints list in the right pane.
Screenshot this: The blue breakpoint marker on the line in the code editor, and the entry in the Breakpoints list.
Step 4
Pause the Code with a Breakpoint
With the breakpoint set, click the button below. DevTools will pause execution on the breakpoint line before it runs. The page will freeze briefly — that is normal. You will see the current line highlighted in the Sources panel.
Screenshot this: The Sources panel paused on the highlighted breakpoint line, with the blue execution marker on the left side of the line.
Step 5
Check Variable Values
While paused, DevTools gives you three ways to inspect variable values.
Scope Pane
In the right pane, open the Scope section. It shows every variable currently in scope at the paused line. Look for: total, alert, and alerts. You will see alert.severity is a string like "high" — that is why adding it to a number produces NaN.
Screenshot this: The Scope section expanded showing the local variables including alert and its properties.
Watch Expressions
In the right pane, click the Watch section and hit the + button. Add these two expressions one at a time:
alert.severity
alert.score
DevTools evaluates both in real time. You will see alert.severity returns a string and alert.score returns the correct number we should be using.
Screenshot this: The Watch section showing both expressions and their evaluated values side by side.
The Console
While still paused, switch to the Console tab. You can type expressions directly and they evaluate in the current scope. Try typing:
alert.score
typeof alert.severity
The first returns the correct numeric score. The second returns "string" — confirming that severity is not a number and cannot be summed.
Screenshot this: The Console with both expressions typed in and their return values visible.
Step 6
Apply the Fix
Resume execution by pressing the blue play button in DevTools (or F8), then remove the breakpoint by clicking the marker again. The fix is to change alert.severity to alert.score inside the loop. The corrected function is already available as calcTotalScoreFixed() in script.js. Click below to run it and confirm it returns the correct total.
Before (buggy)
alerts.forEach(function(alert) {
  total += alert.severity;
});
After (fixed)
alerts.forEach(function(alert) {
  total += alert.score;
});
